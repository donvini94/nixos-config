#+TITLE: NixOS Configuration
#+DESCRIPTION: Flake-based NixOS config for a desktop (dracula) and home server (alucard)

* Overview

Flake-based NixOS configuration managing two systems:
- *dracula* — Desktop workstation (AMD CPU, NVIDIA GPU, Hyprland + caelestia-shell)
- *alucard* — Home server (Hetzner OpenStack VM, nginx reverse proxy, Docker media stack)

Shared base config in =configuration.nix=, per-user tooling via home-manager in =home.nix=.

For detailed architecture, module descriptions, and file-level documentation, see [[file:CLAUDE.md][CLAUDE.md]].

* Build Commands

#+begin_src bash
# Desktop
sudo nixos-rebuild switch --flake .#dracula

# Server
sudo nixos-rebuild switch --flake .#alucard

# Test build (no switch)
sudo nixos-rebuild build --flake .#dracula

# Update flake inputs
nix flake update

# Check / show flake
nix flake check
nix flake show
#+end_src

* Secret Management

Uses [[https://github.com/Mic92/sops-nix][sops-nix]] with age encryption. Three recipient keys (desktop, server, macbook).

#+begin_src bash
sops secrets/dmbs.yaml    # Edit encrypted secrets
#+end_src

Config files: [[file:.sops.yaml][.sops.yaml]], [[file:secrets/secrets.nix][secrets.nix]], [[file:secrets/dmbs.yaml][dmbs.yaml]]

* Adapting This Config

1. Fork and clone
2. Replace =hosts/dracula/default.nix= with your hardware config (or generate via =nixos-generate-config=)
3. Edit =flake.nix= to point at your host
4. Enable/disable modules in =modules/desktop.nix= as needed
5. For server: adapt =hosts/alucard/= — each file is self-contained (networking, services, media, users)
6. Set up your own sops-nix age keys in =.sops.yaml= and re-encrypt secrets

See [[file:CLAUDE.md][CLAUDE.md]] for detailed architecture notes and module documentation.
